\chapter{L.\+EIC Schedules Management System}
\hypertarget{index}{}\label{index}\index{L.EIC Schedules Management System@{L.EIC Schedules Management System}}
\label{index_md_README}%
\Hypertarget{index_md_README}%
 The L.\+EIC Schedules Management System project was developed for the Algorithms and Data Structures course in the 2023/24 academic year of the 2nd year of L.\+EIC at FEUP.\hypertarget{index_autotoc_md1}{}\doxysection{\texorpdfstring{Project Description}{Project Description}}\label{index_autotoc_md1}
Elaborating schedules for L.\+EIC classes can be a complex task. The purpose of this project is not the creation of the schedules, but rather the development of a system to manage schedules after they have been elaborated. The system must include various functionalities related to schedules, such as modifying, searching, viewing, sorting, listing, among others.\hypertarget{index_autotoc_md2}{}\doxysection{\texorpdfstring{Dataset}{Dataset}}\label{index_autotoc_md2}
The project uses a provided dataset available in {\ttfamily schedule.\+zip}, which contains real information about L.\+EIC\textquotesingle{}s schedules for the 1st semester of the academic year 2022/2023 with anonymized student data. The dataset is split into three CSV files\+:


\begin{DoxyEnumerate}
\item {\ttfamily classes\+\_\+per\+\_\+uc.\+csv}\+: Contains the existing classes in each course unit (UC).
\item {\ttfamily classes.\+csv}\+: Contains the schedules of classes.
\item {\ttfamily students\+\_\+classes.\+csv}\+: Contains the classes of the students in each UC.
\end{DoxyEnumerate}

The dataset provides information about classes, students, and their schedules, which is essential for the functionality of the system.\hypertarget{index_autotoc_md3}{}\doxysection{\texorpdfstring{Classes}{Classes}}\label{index_autotoc_md3}
\hypertarget{index_autotoc_md4}{}\doxysubsection{\texorpdfstring{Uc Class}{Uc Class}}\label{index_autotoc_md4}
The {\ttfamily \doxylink{classUc}{Uc}} class represents a course unit (UC) and provides methods for managing UC information.\hypertarget{index_autotoc_md5}{}\doxysubsection{\texorpdfstring{Student Class}{Student Class}}\label{index_autotoc_md5}
The {\ttfamily \doxylink{classStudent}{Student}} class represents a student and provides methods for managing student information, including their schedule.\hypertarget{index_autotoc_md6}{}\doxysubsection{\texorpdfstring{Lecture Class}{Lecture Class}}\label{index_autotoc_md6}
The {\ttfamily \doxylink{classLecture}{Lecture}} class represents a class lecture and includes information about the UC, class code, students enrolled, day, start time, duration, and lecture type.\hypertarget{index_autotoc_md7}{}\doxysubsection{\texorpdfstring{Script Class}{Script Class}}\label{index_autotoc_md7}
The {\ttfamily \doxylink{classScript}{Script}} class reads and processes data from CSV files, allowing the system to handle students and their schedules.\hypertarget{index_autotoc_md8}{}\doxysubsection{\texorpdfstring{Request Class}{Request Class}}\label{index_autotoc_md8}
The {\ttfamily \doxylink{classRequest}{Request}} class handles various operations related to student enrollments, including adding, removing, and switching courses and classes.\hypertarget{index_autotoc_md9}{}\doxysection{\texorpdfstring{Results}{Results}}\label{index_autotoc_md9}
The program allows for the registration and management of various entities, making use of both linear (vector, list, stack, queue) and hierarchical data structures (binary search tree). Important information is saved in files for future use. The program also includes documentation for the code, generated using Doxygen, and indicates the time complexity of the most relevant functions and algorithms.\hypertarget{index_autotoc_md10}{}\doxysubsection{\texorpdfstring{To run the project, run the following commands\+:}{To run the project, run the following commands:}}\label{index_autotoc_md10}
